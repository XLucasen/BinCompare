# 快速开始指南

## 安装和运行

### 1. 编译项目
```bash
cd i:\LucasCSharp\Lisson\App\BinCompare
dotnet build
```

### 2. 运行应用程序
```bash
dotnet run
```

## 首次使用

### 步骤1：加载文件
1. 点击"选择文件A"按钮
2. 在文件对话框中选择第一个.BIN文件
3. 点击"选择文件B"按钮
4. 在文件对话框中选择第二个.BIN文件

### 步骤2：查看对比结果
- 左列显示文件A的内容
- 右列显示文件B的内容
- 有差异的行会用浅红色背景高亮
- 右侧差异信息区列出所有差异

### 步骤3：导航和分析
- 点击差异信息区的地址可快速跳转
- 使用滚动条浏览文件内容
- 两个文件内容会自动同步滚动

### 步骤4：切换显示模式
- 点击"切换为..."按钮在十六进制和二进制间切换
- 从下拉菜单选择每行显示的字节数

### 步骤5：导出结果
- 点击"导出差异"按钮
- 选择保存位置
- 生成差异报告文件

## 测试数据

项目包含两个测试文件：

### 创建测试文件
```powershell
# 文件A：Hello World! + 4个零字节
[byte[]]@(0x48, 0x65, 0x6C, 0x6C, 0x6F, 0x20, 0x57, 0x6F, 0x72, 0x6C, 0x64, 0x21, 0x00, 0x00, 0x00, 0x00) |

  Set-Content -Path 'TestFileB.bin' -Encoding Byte
```

## 常见操作

### 查找特定差异
1. 在差异信息区找到要查看的差异
2. 点击地址（蓝色下划线文本）
3. 自动滚动到该位置

### 切换显示格式
- **十六进制**：显示为大写的0-9、A-F，便于理解二进制数据
- **二进制**：显示为8位的0和1，便于分析位级差异

### 调整行长度
- 8字节：适合详细查看小段数据
- 16字节：平衡显示和可读性（默认）
- 32字节：一次查看更多数据
- 64字节：适合大文件快速浏览

### 清除并重新开始
- 点击"清除"按钮或按Delete键
- 所有数据和对比结果将被清除
- 可以加载新的文件进行对比

## 键盘快捷键速查表

| 快捷键 | 功能 |
|--------|------|
| Ctrl+O | 打开文件A |
| Ctrl+Shift+O | 打开文件B |
| Ctrl+H | 切换显示模式 |
| Ctrl+E | 导出差异 |
| Delete | 清除所有 |

## 故障排除

### 问题：文件加载失败
**解决方案**：
- 确保文件是有效的二进制文件
- 检查文件是否被其他程序占用
- 尝试使用绝对路径

### 问题：显示不完整
**解决方案**：
- 调整窗口大小
- 减少每行显示的字节数
- 尝试不同的显示模式

### 问题：滚动不同步
**解决方案**：
- 这是正常的初始化行为
- 在加载文件后滚动应该会同步
- 尝试重新加载文件

## 性能提示

- 对于大文件（>100MB），应用程序可能需要较长时间加载
- 减少每行显示的字节数可以提高滚动流畅性
- 导出大量差异时可能需要等待几秒钟
